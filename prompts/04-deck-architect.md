# Prompt 04: Deck Architect

Create a deck architecture from the approved source packet and file specification.

Do not render slides yet.

For each slide, produce:

- slide number
- claim headline
- purpose in the argument
- source IDs
- numbers and date ranges
- chart or visual needed
- speaker-note evidence
- assumptions
- open questions
- review status

Rules:

- Headlines are claims, not topics.
- Every claim has source IDs or is marked `unsupported`.
- Every number has source ID and date.
- Speaker notes carry the evidence layer.
